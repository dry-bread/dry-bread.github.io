vue初体验

今天简单的看了看vue.js，说一说自己的简单的体会。
首先是和react对比，vue会稍微更接近原生的写法，但是他们都有组件封装的概念。vue的组件封装会更加复杂一点，比如vue 的组件需要注册后再使用，还有局部封装和全局封装的不同方式，而react则是简单的组件封装方式，几乎全部基于jsx和js原生的方式去写，react会更加有编程的整体逻辑感，对于vue来说，复杂程度我猜测是比react更加复杂的，而vue的数据结构更好，让我能够很容易的去设计一些比较友好和可读的程序。但是我现在还不清楚有什么比较好用的vue的组件库，或者说是免费的，可以共用的组件库，需要自己去尝试，因为今天我发现elementui不是很好用，或者说是设计的不够好。

vue框架笔记

子组件与父组件的沟通

在vue中，我看到了$emit（function，params。。。）方法，该方法可以让子组件去发送某个事件，让父组件去监听到，并且还能抛出参数。

react 子组件与父组件：子组件与父组件是不能直接互相通信的，子组件可以读取到父组件传入的参数，但是不能修改，而父组件无法知道子组件的状态。基于这种情况，就有了状态提升和useContext，状态提升是将关键数据提升到父组件当中，父组件写一些修改数据的方法，将这些方法作为参数传递给子组件，使得子组件有方法去修改父组件的值，我猜测，这里面应该是闭包。或者定义一个“全局变量”（Context）使得所有在内部有所声明static的组件可以共享这个变量，这样子组件与父组件，或者子组件与子组件都能去找到同一份数据。
